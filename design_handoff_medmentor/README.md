# Handoff: MedMentor RAG — клинический тренажёр для студентов-медиков

## Overview
MedMentor RAG — веб-приложение для тренировки клинического мышления. Студент ведёт живой диалог с виртуальным пациентом, собирает анамнез, назначает обследования, формулирует диагноз по ICD-10. Каждый ход системы подкреплён RAG-подсказками со ссылками на клинические рекомендации Минздрава РФ, РКО, приказы и руководства профессиональных обществ.

Пакет содержит дизайн 4 экранов:
- **Landing** — посадочная страница с логином
- **Case List** — библиотека кейсов (домашняя страница после логина)
- **Case Chat** — рабочий экран диалога с пациентом и постановки диагноза
- **Profile** — личный кабинет и настройки

## About the Design Files
Файлы в этом пакете — **дизайн-референсы, сделанные на HTML + React (Babel standalone) + GSAP**. Это прототипы, демонстрирующие визуал и поведение, **а не production-код для прямого копирования**. Задача — **воссоздать эти дизайны в целевой кодовой базе**, используя её окружение (React/Next.js, Vue, нативный и т.п.) и существующие паттерны/библиотеки. Если окружение ещё не выбрано — используйте React + TypeScript + Vite/Next.js как естественную основу (прототипы написаны на React).

Стилизация в прототипах — inline `style={{...}}` и одиночные `<style>`-теги. В продакшене используйте привычный инструмент: CSS Modules, Tailwind, Panda CSS, styled-components и т.п.

## Fidelity

## Screens / Views

### 1. Landing.html
**Purpose:** первое касание с продуктом + точка входа (логин).

**Sections (в порядке сверху вниз):**
1. **Header** (sticky) — логотип MedMentor RAG, nav: Продукт / Как работает / FAQ, справа — ОДНА кнопка:
   - Если НЕ залогинен (`localStorage.mm_logged_in !== '1'`): `Войти` → открывает `LoginModal`.
   - Если залогинен: `Начать кейс` → переход на `Case List.html`.
   - Оба варианта — primary (teal fill, белый текст, стрелка, pill 999, shadow primary).
2. **Hero** — крупный Fraunces-заголовок по словам («Тренируйте *клиническое* мышление на виртуальных пациентах.»), eyebrow-бейдж «КЛИНИЧЕСКИЙ ТРЕНАЖЕР НА БАЗЕ LLM + RAG», одна CTA-кнопка (та же логика: `Войти в аккаунт` / `Начать кейс`), соц-доказательство с аватарами. Справа — `HeroVisual`: стилизованная карточка виртуального пациента.
3. **Stats** — счётчики (GSAP count-up при скролле): 240+ кейсов, 12 специальностей, 8 вузов, 96% рекомендуют.
4. **Features** (`#features`) — 3 фичи, каждая с eyebrow 01/02/03, названием, описанием и визуалом:
   - 01 Диалог с пациентом
   - 02 RAG-подсказки (ЭКГ в 12 отведениях / ESC · тропонин I / РКО · GRACE/TIMI / NICE)
   - 03 Разбор диагностики: сравнение «Ваш ответ / Эталон», 4 критерия с прогресс-барами (Сбор анамнеза 85, Логика 80, Коммуникация 76, Точность 90), итог 83/100, цитата модели в мятной карточке.
5. **HowItWorks** (`#how`) — 4 шага на тёмном фоне (ink), прогресс-линия соединяет центры кружков. Кружки отцентрированы в колонках; линия: `left/right: calc((100% - 96px) / 8)`.
6. **FAQ** (`#faq`) — 6 вопросов, accordion, первый открыт. Ответы отсылают к Минздраву РФ / РКО / приказу №203н.
7. **Footer** — тёмный, только лого + описание проекта + копирайт · «Условия использования» · «Конфиденциальность». Колонки ссылок удалены намеренно — на лендинге нет несуществующих страниц.

**LoginModal** (`landing-modal.jsx`):
- Backdrop `rgba(10,31,31,0.5)` + `backdrop-filter: blur(6px)`.
- Карточка 440px, radius 20, белая. Анимация входа: backdrop opacity 0→1 (220ms); карточка `y:20, opacity:0, scale:0.985` → full (380ms, power3.out). **Критично:** начальные `opacity:0` проставлены inline в style карточки и backdrop, иначе при маунте есть кадр с полностью видимой модалкой до того, как GSAP применит `fromTo`.
- Шапка: teal-иконка 44×44 (monogram M + «+»), h2 «Вход в *MedMentor*», подзаголовок.
- Форма: **только два поля — Логин и Пароль**. Лейблы — uppercase Geist Mono 10.5px. Под полем пароля — ссылка «Забыли?». Кнопка submit teal, 46px, с стрелкой.
- **Не содержит**: переключателя ролей, кнопки «Продолжить как гость», ссылки «Создать аккаунт». Только форма + Войти.
- Ошибка валидации (пустые поля) — shake-анимация через CSS keyframes `mm-shake`.
- На успех (мок — setTimeout 700ms): `localStorage.setItem('mm_logged_in', '1')`, закрыть модалку, перейти на Case List.

## Interactions & Behavior

### Global
- **localStorage keys:**
  - `mm_logged_in` = `'1'` после успешного входа. Landing Header / Hero читают это при маунте и переключают CTA.
- **Навигация:** простые `window.location.href` между HTML. В реальной кодовой базе — роутер (React Router / Next App Router).

### Animations
- GSAP + ScrollTrigger на лендинге: hero word-by-word fade-up, stats count-up, reveal секций при скролле, parallax на декоративных blob-ах.
- Easing по умолчанию: `power2.out` / `power3.out` / `sine.inOut` для флоатов.
- Длительности: 0.22–1.1s.
- Хедер: на `window.scrollY > 10` включает `background: rgba(244,247,247,0.88)` + `backdrop-filter: blur(10px)` + borderBottom.

### States
- Кнопки: hover — лёгкое приподнимание / затемнение; focus — teal outline.
- Поля ввода: focus — `border-color: teal`; error — `border-color: danger`.
- Модалки: click на backdrop = close; Esc = close.

### Validation
- LoginModal: пустые поля → shake-анимация 400ms (CSS keyframes `mm-shake`).
- Diagnosis modal: ICD-10 обязателен, формулировка обязательна.

### Responsive
Прототипы оптимизированы под ≥1200px. Progressive hiding элементов TopBar на Case Chat описан выше. Для продакшена — добавьте мобильные лейауты (sidebar → drawer).

## State Management
Минимально необходимое (React useState в прототипах; в продакшене — Zustand/Redux/Context или server state):
- `isLoggedIn` — глобально, читается из localStorage.
- `loginOpen` — локальное состояние Landing.
- `collapsed` (sidebar) — Case List + Profile + Case Chat, можно персистить в localStorage.
- `activeNav` — текущий пункт меню.
- **Case Chat:**
  - `messages` — массив сообщений диалога (role, text, timestamp, emotion, rag?).
  - `turnCount`, `vitals`, `hypotheses` (массив с процентами), `checklist` (булевы флаги).
  - `diagnosisModalOpen`, `endCaseModalOpen`.
  - Поля формы диагноза.
- **Data fetching (в продакшене):**
  - `GET /cases` — список кейсов.
  - `GET /cases/:id` — детали кейса + паспорт пациента.
  - `POST /cases/:id/messages` — отправка хода (возвращает ответ пациента + опциональные RAG-хиты).
  - `POST /cases/:id/diagnosis` — подтверждение диагноза (возвращает разбор).
  - `GET /me` — профиль.
  - `GET /me/stats` — статистика.
